# Remove Unused devDependencies Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove four confirmed-unused devDependencies (`eslint-plugin-node`, `ts-node`, `standard-version`, `playwright-core`) from `package.json` and `pnpm-lock.yaml` without breaking lint, dev, build, or e2e flows.

**Architecture:** Four independent, atomic removal tasks plus one final integration-smoke task. Each removal task follows the same five-step pattern: (1) grep-verify absence of references, (2) run `pnpm remove`, (3) verify `package.json` diff, (4) run the relevant verification command, (5) commit. Atomic commits allow per-package rollback via `git revert <sha>` if a regression surfaces later.

**Tech Stack:** pnpm 9+ (only package manager — never npm/yarn in this repo), ESLint 8 + `eslint-config-standard@17`, Playwright 1.28, Husky 9, Conventional Commits (enforced by commitlint).

**Scope boundaries (what this plan does NOT do):**
- Does not touch runtime `dependencies` (all 48 are in use per the 2026-04-19 audit).
- Does not address the separate concerns surfaced during audit: missing `vue-tsc` in devDeps, `typescript ^6.0.2` version choice, `.husky/pre-commit` using `npm run lint` instead of `pnpm lint`. Those belong in a separate plan.
- Does not update `CLAUDE.md` — nothing in CLAUDE.md references any of these four packages.

**Only two files are modified across the entire plan:**

## Task 1: Remove `eslint-plugin-node`

**Why it's unused:** `.eslintrc` does not reference `plugin:node/*` in `extends` or `node/*` in `rules`. The `eslint-config-standard@17` preset that `.eslintrc` does extend migrated from `eslint-plugin-node` to `eslint-plugin-n` (a maintained fork); `eslint-plugin-n` is pulled in transitively by the config. The explicit `eslint-plugin-node` declaration in `devDependencies` is a leftover from before that upstream migration.

**Files:**
- Modify: `package.json` (devDependencies block, line ~112)
- Modify: `pnpm-lock.yaml` (regenerated by pnpm)

- [ ] **Step 1: Verify no references in configs or source**

Run (from repo root):
```bash
grep -rn "eslint-plugin-node\|plugin:node/\|\"node/" .eslintrc .stylelintrc commitlint.config.js .husky/ src/ scripts/ vite.config.ts tsconfig.json 2>/dev/null
```
Expected output: **no matches** (empty output, exit code 1 from grep is fine).

Run a second confirmation for Vue/TS source:
```bash
grep -rn "eslint-plugin-node" src/ scripts/ --include="*.ts" --include="*.js" --include="*.vue" --include="*.mjs" --include="*.cjs"
```
Expected: **no matches.**

If either grep produces matches, STOP — the package is referenced somewhere the audit missed; investigate before removing.

- [ ] **Step 2: Remove the package**

```bash
pnpm remove eslint-plugin-node
```
Expected: `package.json` loses one line in `devDependencies`; `pnpm-lock.yaml` regenerates; pnpm reports something like `- eslint-plugin-node 11.1.0`.

- [ ] **Step 3: Verify the diff is exactly one package.json line + lockfile**

```bash
git diff --stat package.json pnpm-lock.yaml
```
Expected: both files changed, only `package.json` and `pnpm-lock.yaml` appear. If any other file changed, something is wrong — revert with `git checkout -- .` and stop.

Then confirm the removed line:
```bash
git diff package.json
```
Expected: one red line matching `-    "eslint-plugin-node": "^11.1.0",` and nothing else.

- [ ] **Step 4: Run lint to confirm ESLint still works**

```bash
pnpm lint
```
Expected: same exit code and warning count as before the removal. If `pnpm lint` now errors with `Cannot find module 'eslint-plugin-node'` or `Failed to load plugin 'node'`, STOP — the package IS referenced somewhere grep missed (possibly a nested config). Revert with `git checkout -- package.json pnpm-lock.yaml && pnpm install` and re-audit.

- [ ] **Step 5: Commit**

```bash
git add package.json pnpm-lock.yaml
git commit -m "chore(deps): remove unused eslint-plugin-node

eslint-config-standard@17 migrated to eslint-plugin-n upstream;
.eslintrc does not reference plugin:node/* or node/* rules."
```

The only hits for the string `ts-node` in the repo are inside compiled sidecar bundles (`sidecar/antares-server.cjs`, `sidecar/antares-server.js`) as transitive package-metadata strings from upstream packages — not actual runtime usage.

**Files:**
- Modify: `package.json` (devDependencies block, line ~126)
- Modify: `pnpm-lock.yaml`

Expected: **no matches.**

Note: `grep ts-node sidecar/*.cjs` will match inside the pre-built sidecar bundle — those are embedded `package.json` metadata strings from transitive deps and are NOT real usages. Don't be alarmed.

## Task 4: Remove `playwright-core`

**Why it's unused:** The e2e test suite imports from `@playwright/test` (not `playwright` or `playwright-core`). The three Playwright packages in devDeps are: `@playwright/test@~1.28.1` (what tests import), `playwright@~1.28.1` (pulled in as peer), and `playwright-core@^1.59.1`. The version mismatch alone is suspicious — `playwright-core` at 1.59 cannot coexist cleanly with Playwright 1.28 since `@playwright/test@1.28` depends on its own matching `playwright-core`. Removing the explicit `playwright-core` entry lets the correct transitive version resolve.

**Files:**
- Modify: `package.json` (devDependencies block, line ~118)
- Modify: `pnpm-lock.yaml`

- [ ] **Step 1: Verify no direct imports of playwright-core**

```bash
grep -rn "from 'playwright-core'\|require('playwright-core')\|from \"playwright-core\"" tests/ e2e/ src/ scripts/ 2>/dev/null
```
Expected: **no matches.** Tests import from `@playwright/test`, not `playwright-core`.

Check `playwright.config.*` too:
```bash
grep -n "playwright-core" playwright.config.* 2>/dev/null
```
Expected: **no matches.**

- [ ] **Step 2: Remove the package**

```bash
pnpm remove playwright-core
```
Expected: one line removed from `devDependencies`. `pnpm-lock.yaml` may keep a transitive `playwright-core` entry at a different (1.28.x) version — that is correct and desired.

- [ ] **Step 3: Verify the diff**

```bash
git diff package.json
```
Expected: one red line matching `-    "playwright-core": "^1.59.1",` and nothing else.

Confirm the transitive resolves correctly:
```bash
pnpm list playwright-core
```
Expected: either "no projects matching" at top level OR a nested playwright-core@1.28.x under `@playwright/test`.

- [ ] **Step 4: Run the e2e suite to confirm Playwright still works**

```bash
pnpm test:e2e
```
Expected: Playwright launches the test browsers, runs `tests/app.spec.ts` (and any other `.spec.ts`), and all tests either pass or fail with the same results as before the removal.

If the suite errors with `Cannot find module 'playwright-core'` or version-mismatch errors, revert:
```bash
git checkout -- package.json pnpm-lock.yaml && pnpm install
```
and stop — the transitive resolution needs investigation.

- [ ] **Step 5: Commit**

```bash
git add package.json pnpm-lock.yaml
git commit -m "chore(deps): remove unused playwright-core

@playwright/test is the only Playwright entry imported by tests;
explicit playwright-core@1.59 was version-mismatched and redundant."
```

## Rollback playbook

If at any point you need to undo the entire cleanup:

```bash
# Revert the four commits in one shot (preserves history, safe)
git revert --no-edit HEAD~4..HEAD
pnpm install
```

Or, if the commits have NOT been pushed and you want them gone entirely:

```bash
git reset --hard HEAD~4
pnpm install
```

Partial rollback (one package at a time):

```bash
# Find the specific removal commit
git log --oneline | grep "chore(deps): remove"
# Revert just that one
git revert <sha>
pnpm install
```
